\section{Benchmarks And Evaluation Protocol}

\subsection{PopQA}
PopQA is used as the recurring-memory benchmark. The final result uses a cache-first 500-example recurring-heavy slice with the same small open model throughout. The post-fix artifact is frozen in \texttt{artifacts/popqa\_500/post\_fix/}.

\subsection{Freshness}
The second benchmark is a deterministic bundled \texttt{FreshQA}-style snapshot that is included with the submission materials rather than downloaded from a public benchmark host. The stream is time-ordered and includes stable updates, confirmations, volatile updates, a rollback probe, and forgetting probes. Evidence visibility is causal: only documents whose timestamp is not in the future are visible to the episode.

This transparency matters. We do \emph{not} claim that the current freshness experiment is a public-benchmark run. Instead, we claim that it is a fully specified bundled artifact and report it as such.

\subsection{Metrics}
Across the two experiments we report:
\begin{itemize}[leftmargin=1.5em]
\item answer quality,
\item latency,
\item retrieval calls,
\item memory reads and writes,
\item adaptation and consolidation counts,
\item stale-answer rate on freshness,
\item rollback count and forgetting-probe behavior on freshness.
\end{itemize}
