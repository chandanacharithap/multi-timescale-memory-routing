\section{Freshness: Updated-Knowledge Routing}

The second experiment validates the full multi-timescale claim. The bundled freshness snapshot asks whether the controller can decide when updated knowledge should be retrieved, written to memory, adapted temporarily, or consolidated durably.

This result turns the controller from a recurring-memory router into a true updated-knowledge router. The full router exceeds retrieval-only and memory-only on answer quality, sharply reduces stale-answer rate, uses \texttt{FAST\_ADAPT} meaningfully, keeps \texttt{CONSOLIDATE} rare, and exercises rollback.

The per-update-type breakdown is also interpretable:
\begin{itemize}[leftmargin=1.5em]
\item stable updates are handled by retrieval-backed writes and later memory reuse,
\item confirmations can trigger guarded consolidation,
\item volatile updates rely on temporary adaptation,
\item rollback probes can invalidate a durable answer,
\item forgetting probes remain correct before and after the update stream.
\end{itemize}
