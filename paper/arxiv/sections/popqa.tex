\section{PopQA: The Positive Stable-Recurrence Regime}

PopQA is the clean positive result in the final paper. It isolates a regime where recurrence is stable, reusable support is valuable, and contradiction risk is low. In exactly that setting, delayed utility helps.

The official FLAN-T5-Base run gives a precise result. Retrieval-only, the retrieve gate, the full router, and the calibrated router all achieve 0.290 quality. But the router and calibrated router reduce retrieval calls from 100.0 to 72.2 while issuing 75.0 memory reads and 49.8 memory writes. In contrast, the no-\(V_{\text{future}}\) ablation (\texttt{router\_no\_v\_future}) falls to 0.090 quality with only 36.0 retrieval calls and much weaker memory use. The paired sign tests are equally sharp: the full router is tied with retrieval (\(p = 1.0\)), while suppressing delayed utility is significantly worse (\(p = 1.58 \times 10^{-30}\)).

This is the strongest direct evidence that delayed utility is not merely decorative. When the environment is stable enough for support to be reused safely, future value changes the preferred policy. The router learns to spend retrieval once, write reusable support, and exploit it later without sacrificing quality. The calibrated router preserves that positive regime because its public-suppression logic does not activate here.
